Build the walking-skeleton vertical slice: the owner can sign in at `/admin/login` with the single seeded admin account and reach a protected `/admin` dashboard that performs a real, RLS-scoped read against the live `orders` table. ## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| unauthenticated browser → /admin/* | Must be redirected; no admin content or DB data rendered |
| stale/forged session cookie → server auth check | Must be revalidated against the Auth server, not trusted from the cookie |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-02 | Elevation of Privilege | public sign-up surface | mitigate | No sign-up route exists; single admin created only via out-of-band `scripts/seed-admin.ts` using the service-role key |
| T-04-03 | Information Disclosure | admin credentials in env | accept | ADMIN_EMAIL/ADMIN_PASSWORD/SERVICE_ROLE_KEY live only in `.env.local` (gitignored) and Vercel env; never committed |
