| Boundary | Description |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Tampering | submit-time jersey assembly | mitigate | Name/number re-validated via the Zod schemas on submit before a CartItem is built; submit disabled until required selections exist, so malformed items cannot enter the cart |
| T-02-11 | Spoofing/Info Disclosure | public storefront route | accept | No auth, no PII persisted, no secrets on the page this phase; storefront is intentionally public (customer-facing) |
| T-02-12 | Tampering | crypto.randomUUID during render | mitigate | UUID generated only inside the submit handler (never during render), avoiding hydration-mismatch / duplicate-id issues (Don't Hand-Roll: cart item IDs) |
</threat_model>
